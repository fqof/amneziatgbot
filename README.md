# AmneziaVPN Telegram Bot
​
Telegram-бот для управления VPN-профилями **AmneziaVPN** на основе Amnezia Admin API — с поддержкой мультипрофилей, одноразовых секретных ключей, коротких ссылок, отдельного веб-сервиса и Telegram Mini App.

## Возможности
​
**Для пользователей**
- 🔐 Мультипрофили — каждый пользователь может иметь несколько VPN-профилей
- 📥 Получение конфигурации (строка `vpn://` и `.vpn`-файл)
- 👤 Просмотр статуса каждого профиля (онлайн, трафик, последнее подключение)
- 🗑 Удаление собственных профилей
- 🖥 Просмотр статуса сервера
- 🔑 Секретные ключи — одноразовый ключ для самостоятельного создания профиля через веб-сервис
- 🔗 Короткие ссылки — временные (24 ч) ссылки на конфигурацию
- 🌐 Telegram Mini App — полноценный веб-интерфейс управления профилями внутри Telegram
​
**Для администраторов**
- 🔔 Уведомления о создании/удалении профилей
- ⚙️ Режим «только для администраторов» (`BOT_MODE=admin`)
- 🧑‍💻 Полноценная админ-панель (см. раздел ниже)
​
**Инфраструктура**
- 🛡 Защита веб-сервисов от сканеров и ботов (`security.py`): блокировка по IP, rate limiting, детектирование подозрительных User-Agent и путей
- 🔒 Шифрование чувствительных данных в базе (Fernet)
​

## Админ-панель
​
- 👥 Список пользователей с пагинацией (кликабельные ID → профиль в Telegram)
- 🃏 Карточка пользователя: все профили, трафик, IP, статус, кнопка перехода в профиль Telegram
- 🚫 Бан/разбан пользователей
- ⏸ Включение/отключение отдельных профилей
- 🗑 Удаление профилей с синхронизацией через Amnezia API
- 📊 Статистика: трафик, топ пользователей, онлайн
- 🔍 Поиск по ID или имени профиля
- 📢 Рассылка сообщений всем пользователям
- 📋 Экспорт пользователей в CSV
- 🔎 Просмотр всех пиров Amnezia
- 🔑 Управление секретными ключами пользователей (блокировка/отзыв)

## Запуск и управление
​
Процессы `bot.py`, `web_service.py` и `miniapp.py` управляются единым скриптом `start.sh` через `screen` — он сам определяет, что уже запущено, и не даёт запустить/остановить повторно.
​
```bash
chmod +x /root/amneziatgbot/start.sh
```
​
| Команда | Действие |
|---|---|
| `./start.sh start` | запустить все сервисы (уже запущенные — пропускаются) |
| `./start.sh stop` | остановить все сервисы |
| `./start.sh restart` | перезапустить все сервисы |
| `./start.sh status` | показать состояние всех сервисов |
| `./start.sh attach <bot\|webapp\|miniapp>` | подключиться к консоли сервиса (`Ctrl+A, D` — отключиться, процесс продолжит работу) |
​
Логи каждого сервиса пишутся в `logs/<имя>.log`.
​

## Amnezia Admin API
​
| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/clients` | список всех клиентов и пиров |
| `POST` | `/clients` | создание нового клиента |
| `PATCH` | `/clients` | обновление статуса клиента |
| `DELETE` | `/clients` | удаление клиента |
| `GET` | `/server` | информация о сервере |
| `GET` | `/server/load` | нагрузка (CPU, RAM, диск) |
| `GET` | `/healthz` | healthcheck |
​
## Веб-сервис (`web_service.py`, порт 5001)
​
Публичный сайт для получения конфигурации по одноразовому секретному ключу, а также для раздачи конфигов по коротким ссылкам.
​
| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/` | страница ввода ключа и имени профиля |
| `POST` | `/connect` | активация ключа и получение конфига |
| `GET` | `/c/<slug>` | страница с конфигом по короткой ссылке |
| `GET` | `/api/ping` | пинг до VPN-сервера (ICMP, серверная сторона) |
| `GET` | `/robots.txt` | запрет индексации |
​
Эндпоинт `/connect` требует динамический токен сессии (`X-Dynamic-Token`), который генерируется при загрузке главной страницы и действует 5 минут.
​
## Mini App (`miniapp.py`, порт 5000)
​
| Метод | Путь | Назначение |
|---|---|---|
| `GET` | `/` | главная страница Mini App |
| `GET` | `/api/me` | профили текущего пользователя |
| `POST` | `/api/create` | создать профиль |
| `DELETE` | `/api/profile/<id>` | удалить свой профиль |
| `GET` | `/api/config/<id>` | получить конфиг профиля + короткую ссылку |
| `GET` | `/api/server` | статус сервера (нагрузка, онлайн, пиры) |
| `GET` | `/api/ping` | пинг до VPN-сервера |
| `GET` | `/api/mykey` | получить (или создать) текущий секретный ключ |
| `POST` | `/api/newkey` | создать новый секретный ключ (старый аннулируется) |
| `POST` | `/api/validate_hash` | валидация Telegram initData |
​
Все эндпоинты, кроме `/api/ping` и `/api/validate_hash`, требуют авторизацию через заголовок `X-Telegram-Init-Data` (подпись Telegram WebApp).
​
## Схема базы данных
​
| Таблица | Назначение |
|---|---|
| `users` | Пользователи: `telegram_id`, `banned`, `key_blocked`, `created_at` |
| `vpn_profiles` | VPN-профили: `telegram_id`, `vpn_name`, `peer_id`, `raw_response`, `last_ip`, `disabled`, `via_key` |
| `secret_keys` | Одноразовые ключи: `telegram_id`, `key_value`, `used`, `revoked`, `can_create`, `used_at` |
| `short_links` | Короткие ссылки: `profile_id`, `slug` (живут 24 ч, чистятся автоматически) |
​
Чувствительные поля (`peer_id`, `raw_response`, `last_ip`) хранятся зашифрованными (Fernet). База поддерживает автоматическую миграцию схемы и миграцию из старой таблицы `vpn_users`.
​
## Безопасность
​
- **Шифрование данных** — `peer_id`, `raw_response`, `last_ip` шифруются в базе (Fernet); повреждённые/легаси-значения не приводят к падению, а логируются и возвращаются как есть.
- **TLS-проверка Amnezia API** — включена по умолчанию (`AMNEZIA_VERIFY_SSL=true`); отключайте только для доверенных внутренних адресов с самоподписанными сертификатами.
- **Защита от спуфинга IP** — заголовку `X-Forwarded-For` не доверяют по умолчанию (используется реальный IP соединения); включайте `TRUST_PROXY_HEADERS=true` только если сервисы действительно стоят за проверенным reverse-proxy.
- **WAF-подобная защита** (`security.py`) — блокировка по IP, rate limiting, детектирование сканеров/ботов по User-Agent и запрашиваемым путям, для веб-сервиса и Mini App.
- **Проверка подписи Telegram WebApp** — все чувствительные эндпоинты Mini App проверяют `X-Telegram-Init-Data`; `MINIAPP_DEV_MODE` для отключения проверки предназначен только для разработки.
- **Динамические токены сессии** — форма веб-сервиса защищена одноразовым токеном (`X-Dynamic-Token`) с TTL 5 минут.
